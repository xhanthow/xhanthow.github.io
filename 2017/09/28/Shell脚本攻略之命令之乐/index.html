简介

掌握Unix/Linux命令行可谓是一门艺术，而且得多实践，下面将介绍一些常见而且实用的命令。

Let’s 🐶

用cat进行拼接

cat(concatenate)：用于读取、显示或者拼接文件内容。

  • 用cat读取文件：cat file1 file2 file3
  • 从标准输入中读取：echo xxx | cat - file.txt(- 将作为stdin文本的文件名)

番外篇:

cat -s file：去除多余的空白格(tr也有此功能，后续将会介绍)

cat -n file：显示行号

录制并回放终端回话

script：录制

scriptreplay：回放

对，这样你就会看到诡异的事情发生了～

文件查找和文件列表

find：沿着文件层下向下遍历，匹配符合条件的然后执行相应的操作。

番外篇:

Unix/Linux：文件系统中的每一个文件都有三种时间戳

  • 访问时间(-atime)：用户最近一次访问文件的时间
  • 修改时间(-mtime)：文件最后一次被修改的时间
  • 变化时间(-ctime)：文件元数据（例如权限或者所有权）最后一次改变的时间

并没有创建时间的概念

它们都可以作为find的参数,-：表示小于 +：表示大于

用tr进行转换

tr：可以对来自标准输入的内容进行字符替换、删除以及重复字符压缩，可以将一组字符变成另一组字符。

  • 大小写转换

  • 简单加密解密

  • 用tr删除字符

  • 字符集补集

  • 用tr压缩字符

  • 计算和（很巧妙）

排序、唯一与重复

sort：既可以从特定的文件也可以从stdin中获取输入，并将输出写入stdout。uniq的工作方式和sort一样

uniq：通过消除重复内容，从给定输入中(stdin或命令行参数文件)找出唯一的行。它只能作用于排过序的数据输入，因此uniq要么使用管道要么将排过序的文件作为输入，与sort命令结合使用。

根据扩展名切分文件名

借助%操作符可以轻松将名称部分从”名称.扩展名”这种格式中提取出来

${VAR%.*}：从${VAR%.*}中删除位于%右侧的通配符(前例中的.*)所匹配的字符串。通配符从右向左进行匹配。%属于非贪婪操作,%%属于贪婪操作：它会匹配符合条件的最长的字符串

${VAR#*.}：从${VAR#*.}中删除位于#右侧的通配符(前例中的.*)所匹配的字符串。通配符从左向右进行匹配
